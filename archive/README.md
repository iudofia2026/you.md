# Archive

**Purpose**: Permanent record of completed tasks and events.

---

## Directory Structure

```
archive/
├── README.md                 # This file
└── 2026/
    ├── 01-january/
    ├── 02-february/
    ├── 03-march/
    │   └── event-name.md
    └── ...
```

---

## Archive File Format

Every archived event follows this template:

```markdown
# [Event Name]

**Date Completed**: [Month] [Day], [Year]
**Status**: COMPLETED

## Context

[Brief description - what was this event?]

## Outcome

[What actually happened - update after event]

## Notes

[Follow-up, lessons learned, action items]
```

---

## How Items Get Archived

**Automatic**: When you use `/your-name mark "X" as done`, completed entries are automatically archived here.

**Manual**: You can also manually add entries to preserve completed work.

---

## Why Archive?

- **Track progress** - See what you've accomplished over time
- **Learn from patterns** - Review outcomes and notes
- **Memory** - Permanent record of your journey
